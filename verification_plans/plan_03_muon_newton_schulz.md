# Plan 03: 验证 Muon 混合 Newton-Schulz 迭代的收敛特性

**目标原子**: `738e20ee` — "Muon Optimizer with Hybrid Newton-Schulz"

---

## 1. 声明回顾

论文声称：
1. Muon 优化器使用混合 Newton-Schulz 迭代进行正交化
2. 前 8 步使用系数 `(a,b,c) = (3.4445, -4.7750, 2.0315)` 实现 "rapid convergence"
3. 后 2 步使用系数 `(a,b,c) = (2, -1.5, 0.5)` 将奇异值 "stabilize precisely at 1"

**核心问题**:
- 8+2 方案真的比标准方案收敛更快吗？
- 为什么选这些系数？

---

## 2. Newton-Schulz 迭代背景

广义 Newton-Schulz 迭代形式：

```
X_{k+1} = X_k * (a*I - b*X_k^T*X_k + c*(X_k^T*X_k)^2)
```

- 2阶: `(a,b,c) = (1.5, -0.5, 0)` — 标准 Newton-Schulz
- 3阶: `(a,b,c) = (2, -1.5, 0.5)` — 更快收敛
- V4 快速阶段: `(3.4445, -4.7750, 2.0315)` — 论文自定义系数

---

## 3. 实验设计

### Phase A: 正交化收敛速度对比

**测试矩阵**:
- 维度 d = {64, 128, 256, 512, 1024}
- 条件数 k = {10, 100, 1000, 10000}
- 分布: 正态 / 均匀 / 对数正态

**对比方案**:

| 方案 | 迭代 | 系数 |
|------|------|------|
| NS-2阶 (baseline) | 10 | (1.5, -0.5, 0) |
| NS-3阶 | 10 | (2, -1.5, 0.5) |
| NS-V4-Hybrid | 8+2 | 前8: (3.4445, -4.7750, 2.0315) + 后2: (2, -1.5, 0.5) |
| NS-V4-Full | 10 | (3.4445, -4.7750, 2.0315) |
| NS-V4-Reverse | 2+8 | 前2: (2, -1.5, 0.5) + 后8: (3.4445, ...) |
| SVD-Gold | — | 精确奇异值置1 (理论上界) |

**测量指标**:
1. **正交误差**: `||X^T X - I||_F / sqrt(d)` (每步)
2. **奇异值偏差**: `max_i |sigma_i - 1|` (每步)
3. **Wall-clock 时间**: 每次迭代的实际运行时间

**关键假设检验**:
- H0: V4-Hybrid 的最终正交误差 = NS-3阶 的最终正交误差 (10步后)
- H0: V4-Hybrid 的收敛速度无明显快于 NS-3阶
- 验证: 后2步稳定阶段是否真的将 sigma 稳定在 1 附近

### Phase B: 训练场景模拟验证

**目标**: 在真实的优化器更新轨迹上验证 Newton-Schulz 的影响。

**步骤**:
1. 训练一个小型 Transformer (~1M params)，使用 Muon 优化器
2. 记录每一步更新矩阵的正交性变化
3. 对比三种方案: (a) No NS — 直接使用梯度更新 (b) NS-3阶 (c) V4-Hybrid
4. 指标: training loss, gradient norm, parameter norm, 收敛步数

### Phase C: 系数敏感性分析

**目标**: 理解 V4 快速系数 `(3.4445, -4.7750, 2.0315)` 的敏感性。

**步骤**:
1. 在系数空间 `(a,b,c)` 中进行 grid search
2. 每个系数组合同样运行 10 步 NS 迭代
3. 测量最终正交误差
4. 绘制 "系数 → 误差" 的 heatmap，定位 V4 系数是否在最优区域
